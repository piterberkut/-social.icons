development
-----------

Тут у нас, значит, исходники сета иконок.

Сами иконки - это отдельные SVG-файлы. Обязательно квадратные.

Иконки "стерилизованы" - одна иконка - это один path.

В иконках не используется stroke.

В самих иконках не проставляются никакие цвета.

Все это нужно для того, чтобы можно было красить иконки стилями.

У иконок "простые" имена файлов - кириллица, цифры, тире. Это важно.

Иконки ложатся в папку symbols/ - одна файл, одна иконка.

В корне файл index.html - в нем можно смотреть на иконке.
Вставлятся они тегом IMG/ - для простоты. Размеры не принципиальны.
Тут можно организовать какую-нибудь верстку при необходимости.

Пачку иконок можно обновлять, редактировать в ней иконки, удалять и добавлять новые.
Главное следовать правилам описанным выше.

После каких либо изменений нужно запустить галп (команда gulp) в коре проекта.
Предварительно установить пакеты для ноды ну и саму ноду.

Галп пройдется по этой пачке иконок и соберет символьный спрайт (symbol/use спрайт для векторной графики).
Он попадает в production/symbols/symbols.svg - эту картинку уже используем в плагине.

Саму картинку подключать не надо, она просто должна лежать в папке, так чтобы к ней можно было обратиться.

использовать иконки так:

<svg width="50" height="50">
    <use xlink:href="symbols/symbols.svg?1480661822#scissors"></use>
</svg>

В атрибуте href путь к векторному файлу, затем через гит парамтр можно пробросить рандомную строку для сброса кеша,
и обращение по айдишнику к нужной иконке в спрайте.

Айдишники совпадают с именами файлов в исходнике.

Красить иконку можно так: fill="#FF0000" прямо на теге SVG:

<svg width="50" height="50" fill="#FF0000">
    <use xlink:href="symbols/symbols.svg?1480661822#scissors"></use>
</svg>

Можно вешать класс:

<svg width="50" height="50" class="red">
    <use xlink:href="symbols/symbols.svg?1480661822#scissors"></use>
</svg>

.red {
    fill: #FF000;
}


Посмотреть можно в production/index.html
(запускать на вебсервере, если просто "открыть с помощью", то браузер не сможет добраться до файла через file://


Альтернативный вариант - это включить спрайт прямо в HTML страницы. В таком случае не надо будет прописывать пути до файла.
Примерно так:
(у спрайта поставить display: none);

<body>

  ...

  <svg style="display: none;">
    тут спрайт
  </svg>

  ...

  <svg width="50" height="50">
      <use xlink:href="#scissors"></use>
  </svg>

</body>
